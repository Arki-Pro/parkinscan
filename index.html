ParkScreen

1. Project Core

The Problem: 5.8 million people in India live with Parkinson's disease. Most lack access to specialist neurology centers in urban cities, causing them to get diagnosed only at advanced stages when treatment is less effective.

The Solution: A ₹1,500 smart wearable screening device. It operates as an on-field screening assistant that records vocal changes and hand tremors to catch early markers of Parkinson's right in rural communities.

3. Target Metrics (Clinical & System Balance)

Built as an accessible indicator pipeline, focusing strictly on high-yield clinical consistency and zero-infrastructure limits:

  • Clinical Tracking: Target correlation index r > 0.6 compared directly with medical-standard UPDRS Part III scores.
  • Processing Boundary: Total edge classification inference executes safely within a <500ms time window on the chip.
  • Zero Cloud Reliance: 100% localized processing. Eliminates the requirement for any active internet connection or external cellular grid.

"All metrics evaluate screening-level clinical utility and signal consistency, not diagnostic accuracy. ParkScreen flags — it does not diagnose."

5. Intelligent Engine Architecture

Runs dual mathematical classifiers simultaneously directly on the processing hardware chip to combine interpretability with accurate vector boundaries.

  1. Signal Pre-Processing: Raw inputs are funneled through specialized spectral filters (80Hz high-pass and 8kHz low-pass bands) to isolate biological signals from ambient background noise.
  2. Data Quality Index (DQI Gate): An automated SNR verification layer checks data purity. If the audio or movement signal is corrupted, the system halts execution and requests a re-test instead of computing a flawed estimation.
  3. Dual-Model Ensemble Classification:
    • Model 1 (Logistic Regression): Provides direct, highly explainable mathematical probability vectors.
    • Model 2 (Linear SVM): Formulates margin boundaries optimized for high classification accuracy (~91% on test cohorts).
  4. Weighted Collapse Fusion: The output from both processes collapses into a synchronized score using the empirical fusion calculation:
    Risk_Score = α·P_LR + (1−α)·P_SVM
  5. Explainable System Reports: Outputs a percentage index alongside an explicit confidence tier, noting the clear performance drivers (e.g., "Elevated tremor frequency isolated; voice index remains uniform").

7. Artifact Management (Anxiety & Colds)

The classification pipeline avoids typical user stress errors using explicit verification constraints:

  • Frequency Elimination: Direct audio filtering blocks transient coughing or mucus noise artifacts prior to running calculations.
  • Longitudinal Verification: Transient stress markers naturally stabilize across multiple sessions. Since genuine neurological signals remain consistent, the local database tracks trajectories across multiple visits to isolate permanent indicators from temporary anxiety.
